Coqoon

Coqoon is an Eclipse plugin providing a feature-complete development environment for Coq proof scripts.

Coqoon is developed alongside Kopitiam, an Eclipse plugin for certifying full functional correctness of Java programs.

Features

  • Eclipse integration

    Coqoon works well with other Eclipse plugins — add more to add new features like version control.

  • Automatic project builder

    Automatically rebuild dependencies when proof scripts change, and forget about manual load path configuration.

  • Java-style project organisation

    Structure Coq developments like Java projects, manage project interdependencies with a user interface, and avoid cluttering source folders with .vo files.

  • Compatibility with other tools

    Coqoon projects can be built outside of Coqoon by using automatically-generated Makefiles.

Requirements

Coq

In order to use Coqoon's PIDE support, you will need a development version of Coq, which you'll need to build separately. Once you have an appropriate version of Coq, you'll need to build the PIDEtop plug-in for it. See Carst Tankink's build instructions for more information.

(It is not necessary to build PIDE itself; Coqoon includes a supported snapshot of the PIDE client.)

Java

Coqoon should work with any recent version of the Java virtual machine. See Oracle's Java downloads page.

Eclipse

Coqoon requires version 4.2 (Juno) or later of Eclipse, available from the Eclipse downloads page.

Operating systems

Coqoon is tested regularly on Windows, Mac OS X and Linux.

Installation

  1. Start Eclipse.
  2. Select the Help → Install New Software... menu option.
  3. Paste the line 
    Coqoon - http://www.itu.dk/research/tomeso/coqoon/e42
    into the Work with: text box, and press Enter.
  4. Select the Coqoon → Coqoon feature, and click Next.
  5. Agree to the Coqoon licence agreement, and click Next.

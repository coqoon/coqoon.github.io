Coqoon

Coqoon is an Eclipse plugin providing a feature-complete development environment for Coq proof scripts.

Features

  • Eclipse integration

    Coqoon works well with other Eclipse plugins — add more to add new features like version control.

  • Automatic project builder

    Automatically rebuild dependencies when proof scripts change, and forget about manual load path configuration.

  • Java-style project organisation

    Structure Coq developments like Java projects, manage project interdependencies with a user interface, and avoid cluttering source folders with .vo files.

  • Compatibility with other tools

    All Coqoon projects come with a configure script, so they can be built and modified without needing to use Coqoon itself.

Requirements

Coq

In order to use Coqoon's PIDE support, you'll need Coq 8.5, the first beta release of which is currently available from the Coq downloads page.

Once you've installed Coq 8.5, follow the installation instructions for PIDEtop to add support for PIDE.

Java

Coqoon should work with any recent version of the Java virtual machine. See Oracle's Java downloads page.

Eclipse

Coqoon requires version 4.2 (Juno) or later of Eclipse, available from the Eclipse downloads page.

As the examples are hosted in a Git repository, we recommend a package that includes Git support out of the box, such as the Eclipse IDE for Java Developers.

Operating systems

Coqoon is tested regularly on Windows, Mac OS X and Linux.

Installation

  1. Start Eclipse.
  2. Select the Help → Install New Software... menu option.
  3. Paste the line 
    Coqoon - https://www.itu.dk/research/tomeso/coqoon/e42-pide
    into the Work with: text box, and press Enter.
  4. Select the Coqoon → Coqoon feature, and click Next.
  5. Agree to the Coqoon licence agreement, and click Next.

Updating

New versions of Coqoon are released frequently to add new features, fix bugs, and improve performance. To update Coqoon to the most recent version, select the Help → Check for Updates menu option.

The most recent version of Coqoon is v0.5.0, released on February 10th, 2015; for more information, see the Coqoon changelog.

Coqoon v0.5.0 does not display PIDE goal information correctly; a new release of Coqoon that addresses this, and other, issues is expected to be released by February 12, 2015.

Configuration

Finding the Coq program

Coqoon will search some system folders for Coq, but if it can't find it—or if it finds an older version without PIDE support—you may need to manually specify the path to the coqtop program.

To specify the path, select the Window → Preferences (or Eclipse → Preferences on Mac OS X) menu option, choose the Coqoon category, and then click Browse to choose the correct folder.

Enabling the Coq perspective

The default Eclipse user interface configuration does not show any Coq-specific views. To enable these, select the Window → Open Perspective → Other... menu option, choose Coq, and click OK.

Toggling PIDE annotations

The Coq editor can optionally display commands that are awaiting execution using a different background colour, providing the user with more immediate feedback.

To enable or disable this option, select the Window → Preferences (or Eclipse → Preferences on Mac OS X) menu option, choose the Coqoon → Interface category, and then click the Enable PIDE processing annotations (experimental) checkbox.

Software Foundations

We have modified a version of Benjamin Pierce's Software Foundations course to be compatible with the development version of Coq that Coqoon currently supports. The easiest way to obtain this project is to import it:

  1. Right-click in the Project Explorer and choose Import.
  2. Choose the Git → Project from Git option.
  3. Choose the Clone URI option.
  4. Paste https://github.com/coqoon/Software-Foundations.git into the URI text box, and press Finish.

This process will create a local copy of the Software Foundations repository on your computer; to import the project again, you should use this local copy directly by choosing the Existing local repository option instead of Clone URI.

To fetch updates from the Software Foundations repository and incorporate them into your local copy, open the Git Repositories view (Window → Show View → Other... → Git → Git Repositories), right-click the Software-Foundations entry, and click Pull.
